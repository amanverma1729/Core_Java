2. Literals
What is Literal?

A literal is a fixed value directly written inside a program.

Example
int a = 10;
char ch = 'A';
String name = "Aman";

10
'A'
"Aman"

are literals.

Types of Literals
1. Integer Literal
int a = 100;
2. Floating Literal
double d = 10.5;
float f = 20.5f;
3. Character Literal
char ch = 'A';
4. String Literal
String str = "Java";
5. Boolean Literal
boolean flag = true;
6. Null Literal
String name = null;
3. Data Types
Why do we use Data Types?

A datatype tells:

What kind of value can be stored.
How much memory is required.
What operations can be performed.

Example:

int age = 25;
double salary = 50000.50;
Types of Data Types
1. Primitive Data Types

Store actual values.

2. Non-Primitive Data Types

Store reference/address of objects.

Primitive Data Types
Type	Size	Example

long	8 bytes	100000L

byte a = 10;

Non Primitive Data Types

Examples:

String

Class

Primitive vs Non Primitive
Primitive	Non Primitive
Stores value	Stores address
Fixed size	Dynamic size
Built-in	User-defined
Faster	Slower

4. Variables
What is Variable?

A variable is a container that stores data.

Example
int age = 21;

age is a variable.

Why do we use Variables?

Without variables:

System.out.println(10);
System.out.println(10);

With variables:

int a = 10;
System.out.println(a);
System.out.println(a);

Reusable.

Characteristics of Variables
Must have datatype
Must have unique name
Case Sensitive
Cannot start with number
Cannot use keywords

Valid:

int age;
int studentAge;

Invalid:

int 123age;
int class;
Types of Variables
1. Local Variable

Declared inside method.

class Test {

    void display() {

        int x = 10;

        System.out.println(x);
    }
}
Memory

Stored in Stack Memory.

Scope

Inside method only.

2. Instance Variable (Non Static Variable)

Declared inside class but outside method.

class Student {

    int id;
}
Memory

Stored inside object in Heap Memory.

3. Static Variable

Declared using static keyword.

class Student {

    static String college = "NITRA";
}
Memory

Method Area.

Shared among all objects.

Difference Between Local, Static and Instance Variable
Feature	Local	Instance	Static
Location	Method	Class	Class
Memory	Stack	Heap	Method Area
Default Value	No	Yes	Yes
Scope	Method	Object	Class
Shared	No	No	Yes

6. Methods
What is Method?

A block of code that performs a specific task.

Syntax
returnType methodName() {

}

Example:

void display() {

    System.out.println("Hello");
}
Method Design
access_modifier returnType methodName(parameters)
{
    //logic
}

Example:

public int add(int a,int b)
{
    return a+b;
}
What is Argument?

Values passed during method call.

add(10,20);

10 and 20 are arguments.

What is Return Keyword?

Used to send value back to caller.

return result;

Example:

int add(int a,int b)
{
    return a+b;
}
Static Method
class Demo {

    static void show() {

        System.out.println("Static");
    }
}

Call:

Demo.show();
Non Static Method
class Demo {

    void show() {

        System.out.println("Non Static");
    }
}

Call:

Demo d = new Demo();
d.show();
When to Create Static Method?

When method doesn't use object data.

Example:

Math.sqrt(25);
When to Create Non Static Method?

When method uses instance variables.

class Student {

    int marks;

    void display() {

        System.out.println(marks);
    }
}
When to Pass Arguments?

When input changes every time.

int add(int a,int b)
{
    return a+b;
}

Pass Non Primitive Type as Argument

    String name;
}

class Demo {

    static void display(Student s) {

        System.out.println(s.name);
    }

    public static void main(String[] args) {

        Student st = new Student();

        st.name = "Aman";

        display(st);
    }
}
Return Non Primitive Type
class Student {

    String name;
}

class Demo {

    static Student getStudent() {

        Student s = new Student();

        s.name = "Aman";

        return s;
    }

    public static void main(String[] args) {

        Student st = getStudent();

        System.out.println(st.name);
    }
}
7. Class

A class is a blueprint/template used to create objects.

Syntax

    int id;
    String name;
}

Class itself occupies memory in Method Area when loaded by JVM.
